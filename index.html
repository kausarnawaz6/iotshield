The framework
Seven phases, one provably correct cycle
Each phase is formally characterized, blockchain-anchored, and validated on real IoT data before the next begins.
Phase 1
Device mapping
Continuous discovery and fingerprinting of all IoT assets. Builds the live network graph that powers isolation decisions downstream.
312-node topology validated
Phase 2
Threat modeling
Ontological feature extraction maps device behavior against known attack patterns across 115 statistical dimensions.
115-feature vector space
Phase 3
Anomaly detection
Validated on 6.1M real traffic samples from 9 commercial IoT devices authentically infected with Mirai and BASHLITE botnets.
AUC 1.000 · FNR 0%
Phase 4
Federated learning
Each device trains locally. Only model weights — never raw traffic — leave the device. 9 real IoT clients, converges in Round 1.
AUC 0.9997 · FNR 0.05%
Phase 5
Blockchain trust
Every automated action is committed to a PBFT append-only ledger before execution. 100% consensus, ~20ms latency, tamper-proof.
100% consensus · ~20ms
Phase 6
Self-healing
SDN isolation → digital-twin rollback → PUF hardware attestation → on-chain reintegration. 99.4% recovery in 6.8 minutes on average.
99.4% recovery · 6.8 min avg
Phase 7
Human oversight
Analysts see only escalations, pre-triaged with blockchain evidence attached. Alert fatigue drops from 60% to 0%. 97.9% automation rate.
96.8% workload reduction
